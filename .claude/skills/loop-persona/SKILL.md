# Loop 2 — Persona/Pain-Point Refresh

Update `personas/personas.md` and `personas/hook-bank.md`. Small, evidenced diffs — this is maintenance, not a rewrite.

1. Web-check for new releases: ANAROCK Homebuyer Sentiment (half-yearly), Knight Frank, JLL India, RBI household-finance data. If new numbers supersede persona figures, update them **with source + date**; keep the old figure in a strikethrough or note so drift is visible.
2. Mine audience language: comments/DMs/search queries the user provides. Extract verbatim pain phrases ("possession delay", "carpet vs super built-up") — audience wording becomes hook wording.
3. Read the last 4 weekly scorecards: promote winning hooks (note their metrics inline), retire the bottom third (move to a "Retired" section with the reason — never delete, losers are data).
4. Add new hooks only if they map to a pain code and pass the ≤3s / ≤10-word / specificity rules.
5. If posting-time A/B has concluded, ensure the fixed slots are recorded in personas.md and remove the `[hypothesis]` tag.

Everything stays traceable: no persona claim without a source, no hook without a pain code.
